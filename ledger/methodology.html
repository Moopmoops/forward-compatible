Methodology · The Ledger

How a claim becomes a record

Every entry in the Ledger passes through four stages: ingestion from named sources, normalization to a shared schema, incident resolution across sources, and verification tiering. This document is the spec for each stage, written so any reader can audit the path from source document to public claim.

1 · Ingestion

How sources enter the system

Each source has its own connector. Connectors are independent — one source breaking, changing its API, or going offline never blocks the rest. Connectors run on a per-source schedule and write to a per-source staging area before any normalization happens.

Three ingestion patterns

PatternWhenSources
REST API pull Source publishes a structured API. Highest fidelity, lowest maintenance. ACLED · UCDP · OHCHR · GDELT
CSV download Source publishes annual or quarterly bulk datasets. Stable schema, predictable cadence. V-Dem · HRMI · SIPRI
Scrape + LLM extract Source publishes narrative reports. Structured scrape pulls report URLs and metadata; Claude extraction pulls (country, date, event_type, actor, casualties) into the normalized schema. HRW · Amnesty · State Dept · USHMM · Citizen Lab · EJAtlas
Reviewer in the loop

For LLM-extracted records, the first several hundred extractions per source go through human review against the original source text. Errors recalibrate the prompt. After calibration, sampling continues at a configurable rate.

LLM extraction is fast and cheap; review keeps it honest.

2 · Normalization

The shared record schema

Every source-record, regardless of origin, is mapped to the schema below before entering the incident store. The original record is preserved alongside the normalized form — no data is ever dropped during translation.

// One source-record { "id": "acled-1023456", "source_id": "acled", "source_record_id": "1023456", "source_url": "https://acleddata.com/event/1023456", "ingested_at": "2026-05-14T06:00:00Z", "country_iso": "MMR", "country_name": "Myanmar", "actor_raw": "Military forces of Myanmar", "actor_normalized": "myanmar.state.military", "actor_relationship_to_state": "state_forces", "event_type": "violence_against_civilians", "date": "2026-05-12", "casualties": { "dead": 22, "injured": 31 }, "summary": "Airstrike on village...", "source_bias_disclosure": null, "incident_id": "inc-2026-05-12-mmr-sagaing-villagestrike", "confidence": 0.85 }

The full schema definition with every enum value lives in schema.json. It is versioned: any breaking change to the shape requires a major-version bump and a migration script for existing records.

Entity resolution

Actor names — "Russian forces", "Russian military", "armed forces of the Russian Federation", "Wagner Group" — need to resolve to a canonical actor identity so that incidents and patterns are visible across sources. Same problem for location names across translations.

Approach: embedding-based candidate matching against a canonical actor table; Claude verification on candidates the embedding alone can't separate confidently. The actor table itself is structured (state forces, state-sanctioned non-state, state-tolerated non-state, fully non-state), with the relationship-to-state classification subject to its own provenance chain.

3 · Incident resolution

Linking source-records into incidents

When ACLED, HRW, and OHCHR all report on the same Myanmar airstrike, the ledger creates one canonical incident with three linked source-records — not three duplicate entries, and not one record that hides the others.

  • Candidate matching by location + date + actor + summary embedding, within a configurable window (default ±48h for events).
  • Claude verification pass on candidates: "Are these source-records describing the same incident? Yes / No / Probably / Different incidents in the same area."
  • Canonical incident generated: best-supported date, location, actor, casualty range across sources, full source-record list.
  • If casualty figures across confirming sources differ by >50%, tier flips to disputed and all source figures surface in the public record.

Why preservation matters

Other aggregators often merge source-records and lose the provenance — a casualty figure shows up with no clear path back to which source said what. The Ledger keeps every source-record addressable: an incident record points to its sources, each source has its own URL and methodology disclosure, and a reader can always reconstruct the chain.

4 · Verification tiering

How records are classified

Every published record carries a tier. The tier is computed, not editorial — it follows from the source count, source agreement, and time window of corroboration.

Multi-source

Two or more independent sources, casualty figures agreeing within ±20% (or all sources reporting in non-numeric terms with compatible descriptions), all within a 30-day corroboration window.

Adjudicated

Confirmed by a court of record (ICC, ICJ, ICTY/ICTR, or comparable national court). The strongest tier; supersedes other tiers for the same incident.

Disputed

Multiple sources cover the same incident but materially disagree on facts or casualty count. Tier is preserved as disputed; the disagreement itself is surfaced in the public record.

Single-source

Reported by one source; not yet corroborated. Surfaced anyway, with explicit single-source tag. This is information, not noise — suppressing single-source reports means losing real signal from regions where coverage is thin.

Why we don't average

Averaging diverging casualty estimates produces a number that has no methodological backing — it's not what any source published, and it implies a precision the underlying data doesn't support.

So when sources disagree, the record carries the range and the methodology behind each estimate. The honest finding field in the public record synthesizes what can be said responsibly. Readers see what each source published; they don't get a fabricated consensus.

5 · Bias disclosure

Every source has known shape

Source bias is metadata, not a flaw. Every record carries the bias disclosure of its source. The reader can choose what weight to give a State Department report on US adversaries differently than a State Department report on US allies — and the system surfaces the information needed to make that judgment.

  • US State Department: congressionally mandated; does not cover US conduct.
  • Russian state media: not used as primary source; appears only when reflected through Western coverage of Russian claims.
  • National governments on their own conduct: reported, tagged as such; corroborated against independent sources.
  • NGO investigative reports: generally high-fidelity; mission framing disclosed (e.g., HRW's focus areas, Amnesty's campaign priorities).
  • Court records: adjudicated facts within the court's jurisdiction; political constraints on jurisdiction itself disclosed.
6 · Public corrections log

Everything that changes, recorded

When a number is revised, a record is removed, or a tier is reclassified, the change writes a row to a queryable public log. The page links to this log from every record. Errors are inevitable; the question is whether the record is opaque about them.

What gets logged

  • Casualty figure revised (before/after, source citation, reason).
  • Tier reclassified (multi → disputed, single → multi, etc.).
  • Source-record added or removed from an incident.
  • Incident merged into another, or split apart.
  • Actor classification changed (state forces ↔ state-sanctioned ↔ non-state).
  • Record retracted (with explicit retraction notice preserved at original URL).
7 · Governance

How the project stays honest over time

Methodology lives or dies on whether it survives changes in funding, leadership, and political pressure. The cultural design has to be load-bearing.

  • Open data, open methodology. Every ingestion script, every normalization rule, every dispute scoring weight is public. The data the public sees is the data the public can audit.
  • No government funding. Foundation and reader-supported. Funding sources published.
  • Independent advisory board. Drawn from existing rights organizations, methodologists, and journalists. Reviews scoring weights and bias disclosures.
  • Public corrections log. All changes recorded with reason and timestamp. Reviewable by anyone.
  • Same lens for every state. No country exempt by virtue of geopolitical alignment. This is the test; without it, the record is just another State Department report in different clothes.
